languageService
service in module languageServiceModule

Description

The Language Service fetches all languages for a specified site using REST service calls to the cmswebservices languages API.

Methods

  • getBrowserLanguageIsoCode()

    Uses the browser's current locale to determine the selected language ISO code.

    Returns
    String

    The language ISO code of the browser's currently selected locale.

  • getBrowserLocale()

    determines the browser locale in the format en_US

    Returns
    string

    the browser locale

  • getLanguagesForSite(siteUID)

    Fetches a list of language descriptors for the specified storefront site UID. The object containing the list of sites is fetched using REST calls to the cmswebservices languages API.

    Parameters
    ParamTypeDetails
    siteUIDstring

    the site unique identifier.

    Returns
    Array

    An array of language descriptors. Each descriptor provides the following language properties: isocode, nativeName, name, active, and required. format: 

    [{
  language: 'en',
  required: true
 }, {
  language: 'fr',
 }]

  • getResolveLocale()

    Resolve the user preference tooling locale. It determines in the following order:

    1. Check if the user has previously selected the language
    2. Check if the user browser locale is supported in the system
    Returns
    string

    the locale

  • getToolingLanguages()

    Retrieves a list of language descriptors using REST calls to the smarteditwebservices i18n API.

    Returns
    Array

    An array of language descriptors. Each descriptor provides the following language properties: isocode, name format: 

    [{
  isoCode: 'en',
  name: 'English'
 }, {
  isoCode: 'fr',
  name: 'French'
 }]

  • registerSwitchLanguage()

    Register a callback function to the gateway in order to switch the tooling language

    Set the user preference language in the storage service

    ParamTypeDetails

isoCode:'fr',
name: 'French'
