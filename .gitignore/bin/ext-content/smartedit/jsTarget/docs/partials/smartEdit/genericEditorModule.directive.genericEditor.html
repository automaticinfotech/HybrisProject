genericEditor
directive in module genericEditorModule

Description

Directive responsible for generating custom HTML CRUD form for any smarteditComponent type.

The link function has a method that creates a new instance for the GenericEditor and sets the scope of smarteditComponentId and smarteditComponentType to a value that has been extracted from the original DOM element in the storefront.

Usage

as element:
<generic-editor
       smartedit-component-type="{String}"
       smartedit-component-id="{String}"
       structure-api="{String}"
       structure="{String}"
       content-api="{String}"
       submit="{Function}"
       reset="{Function}">
</generic-editor>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
smartedit-component-typeString

The SmartEdit component type that is to be created, read, updated, or deleted.

smartedit-component-idString

The identifier of the SmartEdit component that is to be created, read, updated, or deleted.

structureApiString

An optional parameter. The data binding to a REST Structure API that fulfills the contract described in the GenericEditor service. Only the Structure API or the local structure must be set.

structureString

An optional parameter. he data binding to a REST Structure API that fulfills the contract described in the GenericEditor service. Only the Structure API or the local structure must be set.

contentApiString

The REST API used to create, read, update, or delete content.

submitFunction

An optional parameter. It exposes the inner submit function to the invoker scope. If this parameter is set, the directive will not display an inner submit button.

resetFunction

An optional parameter. It exposes the inner reset function to the invoker scope. If this parameter is set, the directive will not display an inner cancel button.
