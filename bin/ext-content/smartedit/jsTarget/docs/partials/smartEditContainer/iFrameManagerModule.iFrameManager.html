iFrameManager
service in module iFrameManagerModule

Description

The iFrame Manager service provides methods to load the storefront into an iframe. The preview of the storefront can be loaded for a specified input homepage and a specified preview ticket. The iframe src attribute is updated with that information in order to display the storefront in SmartEdit.

Methods

  • initializeCatalogPreview()

    If an experience is set in the shared data service, this method will load the preview for this experience (such as Catalog, language, date and time). Otherwise, the user will be redirected to the landing page to select an experience. To load a preview, we need to get a preview ticket from an API

  • load(URL, [checkIfFailingHTML], [homepageInPreviewMode])

    This method loads the storefront within an iframe by setting the src attribute to the specified input URL. If this method is called within the context of a new or updated experience, prior to the loading, it will check if the page exists. If the pages does not exist (the server returns a 404 and a content-type:text/html), the user will be redirected to the homepage of the storefront. Otherwise, the user will be redirected to the requested page for the experience.

    Parameters
    ParamTypeDetails
    URLString

    The URL of the storefront.

    [checkIfFailingHTML]Boolean

    Boolean indicating if we need to check if the page call returns a 404

    [homepageInPreviewMode]String

    URL of the storefront homepage in preview mode if it's a new experience

  • loadPreview(homePageOrPageFromPageList, previewTicket)

    This method loads the preview of the storefront for a specified input homepage URL or a page from the page list, and for a specified preview ticket. This method will add '/previewServlet' to the URI and append the preview ticket in the query string.
    If it is an initial load, load will be called with this modified homepage or page from page list.
    If it is a subsequent call, the modified homepage will be called through Ajax to initialize the preview (storefront constraint) and then load will be called with the current location.

    Parameters
    ParamTypeDetails
    homePageOrPageFromPageListString

    The URL of the storefront homepage or a page from the page list for a given experience context.

    previewTicketString

    The preview ticket.

  • setCurrentLocation(URL)

    This method sets the current page location and stores it in the service. The storefront will be loaded with this location.

    Parameters
    ParamTypeDetails
    URLString

    Location to be stored
